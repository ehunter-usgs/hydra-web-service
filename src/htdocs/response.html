Response Format Documentation

All web service endpoints use a similar response format

Properties

data
Either a GeoJSON Feature, or null if an error occured.
error
Whether an Error occurred. true indicates an error, false indicates success.

NOTE: When error is true, an additional property message is included.

message
Error message, if an error occurred.

NOTE: Only set when error is true.

metadata
Object with information about the request.
date
Request time as ISO8601 formatted string
url
The requested URL including any query string parameters

Example Successful Reponse

{
  "data": {GeoJSON Feature},
  "error": false,
  "metadata": {
    "date": "2016-08-02T18:03:26.122Z",
    "url": "/ws/hydra/event.json?huid=50003UWR"
  }
}

Example Error Response

{
  "data": null,
  "error": true,
  "message": "Event not found",
  "metadata": {
    "date": "2016-08-02T18:03:26.122Z",
    "url": "/ws/hydra/event.json?huid=50003UW"
  }
}
